9 /*!
10  * \class Origen::Solver_matrex
11  * \brief MATREX solver class satisfying Origen::solver.
12  * \author William Wieselquist & Aarno Isotalo
13  *
14  * \sa Origen::Solver, kernel_matrex()
15  *
16  * The options for the MATREX solver are:
17  * - "matrexNterm" (int)
18  * - Minimum number of terms in the matrix power series expansion.
19  * - Must be a positive integer.
20  * - Default: 21.
21  * - "materxNshrt" (int)
22  * - Maximum number of nuclides in a decay chain of short-lived nuclides.
23  * - Must be a positive integer.
24  * - Default: 100.
25  * - "matrexSubsteps" (int)
26  * - Number of equidistant substeps to divide the step to.
27  * - Must be a positive integer.
28  * - Default: 1
29  * - "isAdjoint" (bool)
30  * - Is this an adjoint calculation?
31  * - Must be false (MATREX cannot handle adjoint).
32  * - Default: false.
33  * - "solver" (string)
34  * - Which solver are the options for? Used for some consistency checks.
35  * - Must be "matrex".
36  * - Default: "matrex".
37  * - "matrexResultsCutoff" (double)
38  * - Concentrations below cutoff*sum(abs(n_final)) are returned as zero.
39  * - Must be non-negative.
40  * - Default: 0.0
41  * - "tallyEnergyReleased" (bool)
42  * - Whether to add a tally nuclide to collect the amount of energy
43  * released during the depletion step.
44  * - With non zero flux, uses the reaction kappas that include decay
45  * - With zero flux, uses decay q-values.
46  * - Note that if you sum depletion and subsequent decay steps, the
47  * decay energy will be counted twice (as it was already rolled to
48  * the reaction kappas),
49  * - The result is found in status()->get<double>("energyReleased") and
50  * is in [MeV]/[units of n0]
51  * - Default: false.
52  * - THERE MIGHT BE CASES/DATA WHERE THIS FAILS WITH MATREX
53  * (use CRAM if you need to be sure)
54  * - "tallyNumFissions" (bool)
55  * - Whether to add a tally nuclide to collect the number of neutron
56  * induced fissions.
57  * - The result is found in status()->get<double>("numFissions") and is
58  * in the units of n0.
59  * - Default: false.
60  * - THERE MIGHT BE CASES/DATA WHERE THIS FAILS WITH MATREX
61  * (use CRAM if you need to be sure)
62  * - "tallyNumCaptures" (bool)
63  * - Whether to add a tally nuclide to collect the total number of
64  * neutron induced removal reactions (including fissions).
65  * - The result is found in status()->get<double>("numCaptures") and is
66  * in the units of n0.
67  * - Default: false.
68  * - THERE MIGHT BE CASES/DATA WHERE THIS FAILS WITH MATREX
69  * (use CRAM if you need to be sure)
70  * - "tallyAveConcentrations" (bool)
71  * - Whether to add a tally nuclides to collect the average concentrations
72  * of ALL nuclides in the library.
73  * - The results are found in status()->get<double>("ave_<ZZZAAAI>")
74  * and is in the units of n0. <ZZZAAAI> is the extended ZAID of the
75  * nuclide (leading zeroes ommited).
76  * - Default: false.
77  * - THERE MIGHT BE CASES/DATA WHERE THIS FAILS WITH MATREX
78  * (use CRAM if you need to be sure)
79  *
80  * The default values are returned by Solver::get_defaults() if the Solver is
81  * Solver_matrex. They can also be retreived with the static function
82  * Solver_matrex::get_defaults_static(). The default values should not be
83  * hard coded anywhere else than this class.
