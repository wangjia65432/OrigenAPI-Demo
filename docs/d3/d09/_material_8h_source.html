32 /*! \class Material
33  * \brief Container for an ORIGEN depletable material.
34  *
35  * The Origen::Material uses deques because you can pop off the
36  * front element as you write to a file. So the heavy parts of
37  * the material, the transition matrix and number densities,
38  * can be flushed every so often without a full reallocate.
39  *
40  * The internal units are:
41  * - amount of nuclides \f$ {cm^2}/{barn} \f$
42  * - time in \f$ s \f$
43  * - flux in \f$ {\#}/{cm^2 s} \f$
44  * - power in \f$ W \f$
45  *
46  * The accessors use the following terminology:
47  * - 'bos' for begin-of-step
48  * - 'eos' for end-of-step
49  * - 'amount' for amount (cm^s/barn)
50  * - 'numden' for number density (#/barn-cm)
51  * - 'at' for a value at a particular
52  * - 'over' for an average value over a time interval
53  *
54  * There are generally two ways to 'get' something.
55  * - Get value of x as return value of function, v=x(). This is
56  * used when v is a simple scalar or smart pointer.
57  * - Get value of x with argument value of function get_x(T& v). This
58  * is used in all other cases.
59  *
60  * There is only a single way to 'set' something, set_x(const T& v).
61  *
62  * Using set always implies a copy. To manipulate data directly, which
63  * is not recommended, use the getters that return smart pointers,
64  * e.g. trx_over(), amount_at().
65  *
66  */
