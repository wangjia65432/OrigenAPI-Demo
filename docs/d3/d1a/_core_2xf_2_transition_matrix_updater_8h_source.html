13 /*!
14  * \class TransitionMatrixUpdater
15  * \brief update transition matrices with SCALE/AMPX/ORIGEN
16  *
17  * The CoupleSequence performs two distinct operations.
18  *
19  * 1. Create Origen Libraries from scratch (decay & reaction).
20  * This is typically done once at the beginning of depletion, with
21  * options passed to couple
22  * - jadd=1 to add transitions to the system
23  * - nxx=1 to allow zero transitions in the system
24  *
25  * The OrigenLibraryBuilder performs these operations. Once a library has
26  * been created with the builder, then you can use the Updater below.
27  *
28  * 2. Update Transition Matrices with a fixed set of decays/reactions
29  * during depletion. The TransitionMatrixUpdater performs these
30  * operations.
31  *
32  * The data container expected by initialize has the following components
33  * in Type(name): description format
34  *
35  * - DataContainer(data): data to load during initialize
36  * - string(origenLibraryDefinition)
37  * - string(origenFissionYields)
38  * - string(origenAuxAmpxLibrary)
39  * - DoubleList(storageEnergyBounds)
40  *
41  * - DataContainer(initialize): initialization-specific data
42  * -
43  *
44  * - DataContainer(execute): execute-specific data
45  * - string(AmpxLibrary)
46  * - DataContainer(MaterialList)
47  * - size
48  * - '1' Origen::Material (uses flux spectrum on material)
49  * ...
50  *
51  * \sa TransitionMatrixUpdater.cpp for detailed descriptions.
52  *
53  * \par Single-pass initialize/execute/get/finalize:
54  * \code
55  * Scale::Module::TransitionMatrixModule mod;
56  *
57  * Check(mod.validate(dc)); //see above for DataContainer format
58  * mod.initialize(dc);
59  *
60  * mod.execute();
61  *
62  * // get matrices
63  * size_t n = mod.getNumTransitionMatrix();
64  * for(size_t i=0; i<n; ++i){
65  * Origen::SP_TransitionMatrix tmat = mod.getTransitionMatrix(i);
66  * Origen::print_transition_matrix(tmat);
67  * }
68  *
69  * mod.finalize(); //clears all members
70  * \endcode
71  *
72  * \par Multi-pass with update:
73  * \code
74  * Scale::Module::TransitionMatrixModule mod;
75  *
76  * mod.initialize(dc);
77  *
78  * mod.setExecute_AmpxLibrary(matAmpx); //SP
79  * mod.execute(dcA);
80  * // ... get some matrices
81  *
82  * mod.setExecute_AmpxLibrary(matAmpx); //SP
83  * mod.execute(dcB);
84  * // ... get more matrices
85  *
86  * mod.finalize(); //clears all members
87  * \endcode
88  *
89  * \par Use setData* functions to prevent touching disk:
90  * \code

94  * mod.setData_origenYields(dataYields); //SP
