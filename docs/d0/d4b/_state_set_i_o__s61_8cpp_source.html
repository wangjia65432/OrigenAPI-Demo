9 This description has been copied from the SCALE 6.1 Manual, pg. F7.A.1,
10 the section titled "F7.A. DESCRIPTION OF CONCENTRATION BINARY FILES".
11 Only minor formatting adjustments have been made.
12 
13 The concentrations, in units gram-atoms, of all the ORIGEN-S nuclides
14 may be saved to the binary file associated with unit NXTR, by applying
15 the unit number input in the 0$ data or the default of 71 (which makes
16 the file named ft71f001). These concentrations may be read in a
17 subsequent run to define the initial concentrations for a restart case.
18 The input descriptions in Sects. F7.6.3 and F7.6.4 have details of using
19 NXCMP in the 56$ data to apply input from NXTR or using MOUT and MSUB in
20 the 56$ data to save concentrations and spectral data at requested time
21 steps on unit NXTR. There are other uses of unit NXTR, related to
22 creating certain photon libraries, that should not be applied if
23 concentrations are wanted. All of the data associated with a given time
24 step are saved in two binary records. The position of each of these pair
25 of records is dependent upon the order in which the time step is saved.
26 If data at a given position are read from NXTR (e.g., restart case), the
27 first subsequent time step saved will be located in the next position
28 after that of the input time step record. It is recommended that the
29 user copy the file before using it, particularly if data following that
30 input needs to be saved and is to be written over. At the end of a job
31 in which data are saved in NXTR, the first record of the pair described
32 above is written again with the first variable set to zero, making a
33 termination record. The remaining variables are not used. Also, arrays
34 pertaining to spectral data that were not requested by the input would
35 have meaningless data and should not be used. The first record, in the
36 pair discussed above, is always written with the same set of variables.
37 The three Fortran 90 write statements are as follows:
38 
39 \code{.f90}
40 
41  !Record Type 1
42  write (unit=n96) itxx, ilite, iact, ifp, nrflag, msub1, nstepo, &
43  kasepo, jobpos, nocs, noblnd, ndset, ntype, ngrp, nelem, &
44  nvert, ng, mmn, mout, index, mstar, nunit, kblend, nenle, &
45  nenac, nenfp, l1, l2, l3, l4, tmo, fracpw, tconst, tunit, &
46  twrite, pwrite, fwrite
47 
48  !Record Type 2
49  write (unit=n96) (nucl(i),i=1,itot), (x(msub1,j),j=1,itot), &
50  title, basis, (egamle(i1),i1=1,n1), (egamac(i2),i2=1,n2), &
51  (egamfp(i3),i3=1,n3), (specle(i4,msub),i4=1,l1), &
52  (specac(i5,msub),i5=1,l2), (specfp(i6,msub),i6=1,l3), &
53  (ener(i7),i7=1,n4), (dsav(i8,msub),i8=1,ng), &
54  (esav(i9,msub),i9=1,ng), (itsav(i10,msub),i10=1,20), &
55  (spnnuc(i11,msub),i11=1,l4),(alpnuc(i12,msub),i12=1,l4), &
56  (spneut(i13,msub),i13=1,ngrp), & (specan(i14,msub),i14=1,ngrp), &
57  (specsp(i15,msub),i15=1,ngrp),(eneuts(i16),i16=1,n5)
58 
59 \endcode
60 
61 where
62 
63 \code{.f90}
64 
65  n2 = nenac + 1
66  n3 = nenfp + 1
67  n4 = ng + 1
68  n5 = ngrp + 1
69 
70 \endcode
71 
72 The variable and array descriptions below indicate the position of
73 the listed variable, the variable name, the size of an array or a
74 character variable, and the variable‘s definition or meaning. The first
75 record contains only integers, except for one character variable, TUNIT.
76 Most of the values are sizes of libraries, spectrum energy-group sizes, or
77 other array sizes. Some values indicate the data position in the file. Some
78 are code flags or triggers used in specifying data. The second type of record
79 contains only arrays, either numerical or character types, where the sizes are
80 either given or readily obtained (e.g., by adding one) from the first record
81 values. The spectra energy-group structures are in ascending order. The gamma
82 spectra may pertain to all nuclides in the problem, or a subset of nuclides
83 (e.g., actinides only) as determined from the case input description used to
84 generate the spectra.
