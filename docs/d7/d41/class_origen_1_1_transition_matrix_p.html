Add a tally nuclide to the system.

Warning
After calling this function, only add_tally_nuclide, fold_flux, get_non0_/non0/kd-/kd/loc_/loc/nucl are quaranteed to work. All other functions will exhibit unspecified behavior. All classes using this TransitionMatrix will exhibit unspecified behavior.
The modified matrix is only quaranteed to work with CRAM.
Note
The caller is responsible for keeping track of his tallies. The results of the j:th tally added will be output as the concentration of the N + j:th nuclide, where N is the number of real nuclides.

A tally nuclide produces the integral of sum N_i rw_i phi + N_i dw_i over the length of the step. N_i is the concentration of the i:th nuclide and rw_i and dw_i the corresponding weights for decay and transition reactions. Negative weights are permitted, if you figured a use for those.

Parameters

[in]tally_riIndices i of the nonzero reaction weights.
[in]tally_dwThe nonzero weights dw_i specifying the tally.
[in]tally_diIndices i of the nonzero decay weights.
[in]zaidZAID for the tally nuclide. Used for sorting in CRAM. This can have a large effect on performance.
Returns
Index of the freshly added tally in the composition vectors.
